JWT Authentication for Banking API

1. Aim
To implement JWT-based authentication for banking API endpoints.

2. Objectives
● Create user registration and login endpoints
● Generate and verify JWTs
● Protect banking routes
● Implement token refresh mechanisms
● Add account-level security

3. Hardware/Software Requirements
● Node.js 18+
● Express.js
● MongoDB
● jsonwebtoken library
● Postman

4. About the Experiment
This experiment introduces security practices including:
● Password hashing using bcrypt
● JWT signing and verification
● Secure route protection
● Token expiration and refresh
● Securing sensitive user data

Output
● User registration and login functionality
● JWT generation on successful login
● Protected route access using tokens
● Expiry and invalid token handling
● Passwords stored securely in the database
